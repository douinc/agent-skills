# Dou 제품 와이어프레임 가이드

모든 와이어프레임 HTML은 이 가이드를 따른다.

---

## CSS 규칙 (필수)

**Tailwind CSS CDN을 사용한다. `<style>` 블록으로 raw CSS를 작성하지 않는다.**

```html
<script src="https://cdn.tailwindcss.com"></script>
```

- 모든 스타일은 Tailwind 유틸리티 클래스로 작성
- 커스텀 값은 Tailwind arbitrary value 문법 사용: `w-[360px]`, `bg-[#f0f0f0]`, `text-[13px]`
- `<style>` 블록은 `@keyframes` 애니메이션 정의에만 허용
- `font-family` 설정이 필요하면 `body { font-family: ... }` 한 줄만 `<style>`에 허용

---

## 와이어프레임 수정 규칙

피드백을 반영할 때 **요청한 부분과 직접 연관된 곳만** 수정한다.

**수정 전 반드시 확인:**
1. 어느 섹션(1. 기본 상태 / 2. 빈 상태 등)이 영향을 받는가
2. CSS 변경인지 HTML 변경인지 — CSS 변경은 해당 클래스를 쓰는 모든 섹션에 영향을 줄 수 있다

**Edit 도구 사용 규칙:**
- `replace_all: true`는 "모든 섹션에 동일하게 적용해야 하는 변경"일 때만 사용
- 특정 섹션만 바꿀 때는 반드시 섹션 주석(`<!-- 1. 기본 상태 -->`)을 `old_string`에 포함해 범위를 고정
- 수정 후 변경된 섹션을 명시: "1번·2번 섹션의 모바일 프레임 수정됨"

**이 생각이 들면 멈춰라:**

| 생각 | 현실 |
|------|------|
| "같은 클래스니까 replace_all로 한 번에 바꾸면 편하지" | 섹션마다 의도가 다를 수 있다. 범위를 먼저 확인해라. |
| "어차피 비슷한 구조니까 다 바꿔도 되겠지" | 담당자가 요청한 범위만 바꾼다. 나머지는 손대지 않는다. |

| 용도 | 값 |
|------|-----|
| 페이지 배경 | `#f0f0f0` |
| 프레임 배경 (흰 화면) | `#fff` |
| 프레임 테두리 | `#ccc` |
| 구분선 | `#eee` 또는 `#ddd` |
| 주요 텍스트 | `#000` 또는 `#333` |
| 보조 텍스트 | `#666` 또는 `#888` |

| 비활성 요소 | `#aaa` 또는 `#bbb` |
| Primary 버튼 배경 | `#000` |

| 뮤트 배경 | `#f5f5f5` |

## 페이지 구조

```html
<!-- 1. 상단 고정 헤더 (페이지 제목 + 부제목) -->
<div class="page-header"> ... </div>

<!-- 2. 화면 상태별 섹션 (번호 + 제목 + 설명 + 프레임 페어) -->
<div class="screen-section"> ... </div>
<div class="divider"></div>
<div class="screen-section"> ... </div>
...
```

### 페이지 헤더
- 위치: `position: sticky; top: 0; z-index: 100`
- 내용: 페이지 제목 + 부제목 (`그레이스케일 와이어프레임 · 모바일 360px · 데스크탑 820px`)

### 섹션 헤더
```
1. 기본 상태 — 전체 탭        ← 번호. 제목
목록이 있을 때의 기본 화면      ← 한 줄 설명
```

### 섹션 간 구분
- `<div class="divider">` — 1px 수평선

---

## 프레임 페어 (핵심 규칙)

**모든 화면 상태는 모바일(좌) + 데스크탑(우)을 세트로 배치한다.**

```
[섹션 제목]
[모바일 프레임]    [데스크탑 프레임]
```

단독 모바일만, 단독 데스크탑만 배치하지 않는다.

### 모바일 프레임
- 너비: `360px`
- 구성: 상단 헤더 → 콘텐츠 → 하단 액션바(조건부) → 하단 탭바 (제품에 따라)

### 데스크탑 프레임
- 너비: `820px`
- 구성: 사이드바(196px) + 페이지 콘텐츠 (제품에 따라 다를 수 있음)

---

## 프레임 내부 공통 요소

### 고정 요소 표기
고정(fixed/sticky) 요소 아래에 주석 텍스트 추가:
```
※ 고정
※ 고정 (탭바 위)
```

### 하단 액션바 (있는 화면만)
- 모바일: 탭바 바로 위에 배치. `※ 고정 (탭바 위)` 표기
- 데스크탑: 콘텐츠 하단에 배치 (fixed 아님)
- Primary 버튼: `max-width: 260px` (데스크탑)

## 텍스트 규칙

- **실제 UI 문구를 그대로 사용** — Lorem ipsum, "텍스트", "내용" 금지
- 모든 버튼 라벨, 빈 상태 문구, 배지 텍스트는 실제 화면에 들어갈 문구
- 날짜/이름/숫자는 현실적인 예시 사용

## 배지 스타일

| 상태 | 스타일 |
|------|--------|
| 일반 상태 배지 | `border: 1px solid #ddd; color: #666; background: #fafafa` |
| 오류 배지 | `border: 1px solid #f5c0c0; color: #c0392b; background: #fff5f5` |
| 카운트 배지 (탭 옆 숫자) | `background: #000; color: #fff; border-radius: 10px` |

## 빈 상태 구조

```
[아이콘 박스 44×44px, background: #eee, border-radius: 10px]
[제목 — font-size: 14px, font-weight: 600]
[설명 — font-size: 12px, color: #aaa]
[버튼 — 있는 경우만]
```

---

## 섹션 명명 규칙

섹션 제목은 `번호. 상태명 — 조건` 형식:

```
1. 기본 상태 — 전체 탭
2. 목록 탭 — 항목 있음
3. 목록 탭 — 빈 상태
4. 빈 상태 — 데이터 없음
```
